# RSS 当日资讯 → `content/post/YYYY-MM-DD.md`

## 目标

从已配置的 RSS/Atom 源汇总**当日**条目，输出**一篇**按日期命名的 markdown，与本博客 frontmatter 风格一致（TOML `+++`，标题形如 `AI早知道 【YYYY-MM-DD】刊`）。**资讯条目中的标题、摘要等须译为通顺的中文**；产品名、公司名、技术缩写等可保留英文或中英并用，以可读为准。

**输出目录**：`content/post/`。

## 何时运行

- 用户要求「今日 RSS / 抓取资讯 / 生成当日 md」等。
- 编辑 `feeds.json` 之后，可重新运行以刷新内容。

## 流程

1. **目标日期**：默认使用 **`Asia/Shanghai` 时区下的「今天」**。若用户指定日期，则使用该日期（ISO `YYYY-MM-DD`）。
2. **执行抓取脚本**（见下文），在仓库根目录运行。脚本会为每条归入 **要闻 / 模型发布 / 开发生态 / 产品应用 / 技术与洞察 / 行业生态 / 前瞻与传闻 / 其他**（关键词启发式），优先运行脚本以保证输出可复现；仅在脚本无法运行（如网络被拦等）时再手写文件。
3. **检查**生成文件：去重、按需删掉跑题条目，按用户意愿设置 `draft = true` 或 `false`。
4. **中文表述**：定稿前将**每条标题、摘要** 都通过 Agent 译为自然、准确的中文（链接 URL 不变；无摘要可省略「摘要」行）。
5. **不要**粘贴大段受版权保护的全文；RSS 摘要为**短摘录**定稿前可人工删改、润色或译成中文。
6. **不要**抓取和正文内容无关的图片，包括联系方式，二维码等
7. **不要**用脚本翻译，借助Agent自身能力，直接翻译就好，不要问用户了

## 命令

在仓库根目录执行：

```bash
python .cursor/skills/rss-daily-digest/scripts/fetch_rss_digest.py
```

可选参数：

```bash
python .cursor/skills/rss-daily-digest/scripts/fetch_rss_digest.py --date 2026-04-20
python .cursor/skills/rss-daily-digest/scripts/fetch_rss_digest.py --feeds .cursor/skills/rss-daily-digest/scripts/feeds.json
```

依赖：**Python 3.10+**，**仅标准库**（无需 `pip install`）。

## 输出结构

严格按下列 Markdown 骨架输出；**概览与正文**按下述八类归纳（顺序固定；当日无条目的类在概览中写「（暂无）」）：

**要闻** → **模型发布** → **开发生态** → **产品应用** → **技术与洞察** → **行业生态** → **前瞻与传闻** → **其他**

与现有文章一致，例如：
-------------------------------------------------------------------

## [中文标题](链接地址)

- 中文摘要（来自 RSS）

-----

相关链接：
- 「相关链接」下每条**仅一行**，可见文字（方括号内）与链接目标（圆括号内）**均为同一完整 URL 字符串**，例如
  `- [https://example.com/path](https://example.com/path)`。

脚本会按此结构写入；**正文条目的先后顺序与概览八类一致**（每一类内与 RSS 抓取/排序一致）。**定稿 md 须以中文为主**。若用户有要求，可微调小节标题或个别归类以贴合博客语气。


## 订阅源列表

默认精选列表见 [scripts/feeds.json](scripts/feeds.json)。编辑者可增删源（名称 + url）。若某源返回 403 或为空，可换该站其他官方 RSS 地址，或从列表中移除。

更多来源与排错说明见 [reference.md](reference.md)。
